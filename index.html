// Resource Design

リソース設計

各タイトルのリソース種別・入手元・消費先と設計の特徴をまとめたもの。

共通リソース構造
  • 基本リソース層（全ゲーム共通）
  • プレミアム通貨（石・ダイヤ等）: ガチャ・スタミナ回復に使用
  • スタミナ（AP・ライフ等）: コンテンツ消化の制限リソース
  • ゴールド・コイン: 強化・合成素材購入に使用
  • EXP素材: キャラLvアップに使用
  • ゲーム固有の差別化リソース
  • （ウマ娘）スキルpt: 育成完了時に割り振る選択型リソース
  • （ホワサバ）満足度: 住民の要望に応えることで獲得する特殊リソース
  • （キノコ伝説）ランプ: 核心コンテンツに直結する独自リソース
  • （学マス）活動費: PLvで獲得量が変わる成長リソース
タイトル別リソース一覧
ゲーム主要リソース入手元設計のポイント
🐎 ウマ娘スキルpt、サポートカード、スタミナ育成イベント、ガチャスキルptの割り振り選択でプレイヤーに意思決定コストを与え満足感を上げる
📘 ブルアカピロウ（AP）、宝石、各素材クエスト、ショップキャラLv上限をメインストーリー進行度に連動させ課金だけでは解決しない設計
🔥 ヘブバンライフ（AP）、奇跡の鈴、殻石メインストーリー、プリズムバトル交流にライフを消費させることで「コスト感」を生む
⚔️ プリコネマナ、プラチナコイン、イベントアイテムクエスト、クランバトル月次クランバトルで大量消費→再蓄積のサイクルが生まれる
🤖 NIKKE司令官EXP、殻石、各強化素材キャンペーン、タクティクアカデミー面談→好感度→エピソード解放という段階的コスト設計
💀 メメントモリダイヤ、強化水、潜在宝珠、経験珠放置報酬、幻影の神殿供給・消費量バランスが秀逸で常に微不足感がある
🎤 学マスマニー、レッスンノート、活動費プロデュース、アチーブメントPLvを上げることで活動費獲得量アップという成長フィードバック
🏰 ラストウォー耐熱合金、英雄EXP、建造加速施設、荒野、ドローン建造・研究の時間コストが常に発生し「加速」への課金動機を生む
🏴‍☠️ バウンティラッシュ虹のダイヤ、キャラかけら、スキル玉リーグバトル、ウィークリーミッションリーグランクに応じた報酬差がリーグを上げる動機を形成
⚽ ブルーロック各練習素材、スキルブックトレーニング、ガチャ練習メニュー選択でサポートメンバーの能力が変わるため素材選択が重要
🗡️ ホワサバ英雄EXP、鋼材・木材・石炭、満足度探検、荒野、施設満足度という「住民管理」型リソースで施設強化への内発動機を生む
🍄 キノコ伝説ランプ、コイン、神威鎚自動収集、試練自動収集が止まることへの恐怖感が離脱防止に機能
🌀 ファンパレ銅彩/銀彩素材、修練の燈メインクエストGRADEシステムで必要素材を分散しメインクエスト進行を強く促す
👾 怪獣8号次元晶、怪獣鉱、クレジットクエスト、交流ストーリー進行度でコンテンツロック→リソース獲得先が自然に解放
リソース設計の3パターン
⏱️ 時間依存型
  • 放置系ゲームで採用（ラストウォー・メメントモリ等）
  • リソース補充に時間がかかることで「加速」への課金動機を生む
  • 建造・研究・治療の待ち時間が常に存在
  • ログインの継続動機になる（補充されたリソースを使いたい）
🎯 クエスト連動型
  • メインクエスト進行でLv上限・コンテンツ・素材入手先が解放
  • ストーリーを進めることにリソース的な意味を持たせる
  • ファンパレ・ヘブバン・ドラクエタクトで顕著
  • ガチャ（課金）だけでは解決できない進行壁を作れる
🎲 ランダム供給型
  • 育成型ゲームで採用（ウマ娘・ブルーロック・学マス）
  • ランダムイベントでスキル・パラが不確定に変動
  • 「欲しいスキルが出なかった → もう1回」を生む
  • スキルpt等の確定性リソースと組み合わせて不確実性を緩和

// Monetization

マネタイズ設計

観測されたマネタイズスキームと各手法の特徴。

マネタイズ手法一覧
🎰 ガチャ（スタンダード）
  • 欲求の最大化 → 限定性の強調 → 天井保証で安心感
  • 初回10連無料でハードルを下げる
  • 学マス・怪獣8号・ファンパレ等で採用
  • 無料ガチャチケット配布でプレイヤーに課金体験を提供
📬 バトルパス / ミッションパス
  • 月額・週額の少額継続課金
  • 課金ハードルを下げ「継続課金習慣」を形成する
  • バウンティラッシュ・ドット勇者・学マス等で採用
  • 無料帯との差を見せることで課金動機を育成
🔮 後払いガチャ
  • 結果を見てから支払う/キャンセルを選択できるスキーム
  • 「無駄な課金をしたくない」心理を活用
  • 結果確認後のジレンマがむしろ課金率を上げる可能性
⭐ ウィッシュガチャ
  • 指定回数目に選んだキャラが確定排出
  • 「いつかは手に入る」安心感で長期課金を促進
  • ターゲットキャラへの強い愛着がある場合に特に有効
👑 VIPシステム
  • 少額課金から始められる課金ステータスが累積
  • （メメントモリ・ドット勇者）VIPLvが上がるほど恩恵が増大
  • 一度課金すると「もったいない」心理でVIPを維持したくなる
📦 パック販売
  • 特定需要に直接応える高単価購入
  • スタートダッシュパック・期間限定パックが代表的
  • お得感の演出がポイント（通常より〇〇%多い等）

// 育成型

🐎 ウマ娘

「育成」がメインアクションのゲーム。ランダムイベントと特別移籍によるリソースサイクルが特徴的。

コンテンツ・機能一覧
  • 育成
  • 育成ループ — ウマ娘選択 → サポカ編成 → トレーニング → レース出走 → 殿堂入り → 継承
  • └ シナリオ育成（グランドライブ・星振る夜のアリア・Dreams等）で分岐・深化
  • PvPコンテンツ
  • チーム競技場 — 殿堂入り3人チームでRP消費のチームレース。チームランク制
  • チャンピオンズミーティング — チームランクE2解放。月次開催のリーグ制PvP。予選〜決勝ラウンド
  • リーグオブヒーローズ — チャンミ非開催月に隔月開催のPvP（同形式）
  • ルームマッチ — 最大18人のフリー対戦部屋。練習・チャンミ前調整に使用
  • シングルレース — 1人用フリーバトル。フォロー中トレーナーの代表ウマと一緒に出走可
  • その他コンテンツ
  • ストーリーイベント（月次） — 期間限定のイベントストーリー＋レース。限定サポカ入手機会
  • レジェンドレース — 史実の名馬と対決する対NPC高難度レース
  • サークル — プレイヤー間コミュニティ。総獲得ファン数で全国ランキング競争
  • ウイニングライブ — レース後の演出。1着でセンター、衣装変更スキン適用
  • トレーナーガイド — チャンミ向け育成補助。推奨スキル・継承ウマを提案する機能

🗺️ このサイトの使い方

新規ゲーム企画者・既存タイトル施策担当者、どちらにも使えます。目的に応じた2つのルートで案内します。

🎯 Route A：新規ゲームを企画したい
1タイトル比較表を開き、作りたいジャンル・インセンティブタイプを確認。先行タイトルの強みと脆さを把握する

3各タイトルページの類似タイトル分析で「何が同じで何が違うか」の差分を読み、自分のゲームの差別化ポイントを言語化する
4ガチャ設計比較でマネタイズ設計の選択肢と業界標準を確認する
🔧 Route B：既存タイトルの施策を考えたい
1施策パターン辞典で「今打ちたい施策のパターン」と実装のポイントを確認する
2類似タイトルの施策タイムラインで「同じ状況で何を打ったか」の実例を見る

// Onboarding Design

🚪 オンボーディング設計

初回セッションでユーザーをどう定着させるか。チュートリアル構造・初動報酬・最初の感情ピークの設計パターンを分析する。

// オンボーディングの3フェーズ
PHASE 1
最初の30分
「面白そう」→「続けてみよう」
  • チュートリアルで最低限の操作を習得させる
  • 最初の無料ガチャで「当たった」体験を提供
  • メインキャラクターへの感情的接触（声・ストーリー）
  • 初回クリアの達成感を早期に設計
PHASE 2
最初の3日間
「やることがある」状態を作る
  • デイリーミッションの習慣化（初回3日で定着）
  • スタミナ消費サイクルへの誘導
  • 初回課金報酬（初回限定パック等）の提示タイミング
  • ギルド/クランへの参加誘導
PHASE 3
最初の2週間
「このゲームに投資した」感覚
  • 育成・強化の進捗が目に見えて感じられる
  • 対人コンテンツへの初参加（壁の発生）
  • 「もっと強くしたい」欲求の芽生え
  • コミュニティへの帰属感の形成
// タイトル別：初回定着設計の特徴
🐎 ウマ娘 最初の育成1周が自然なチュートリアルになっている。初回育成の感動体験が定着の核
⚔️ FGO 序章ストーリーの完成度が高く「続きが読みたい」動機で自然定着。石配布で最初のガチャ体験
🃏 ポケポケ 開封体験が極めてシンプル。1日2パック無料が初日から習慣化を自動設計
🏰 ラストウォー 基地建設チュートリアルが加速材の使い所を自然に教える。早期の同盟加入誘導が離脱防止
👑 Royal Match 最初のステージ群が簡単に設計されており「自分は上手い」錯覚を作る。部屋改装の達成感が早期に提供
🌍 原神 オープンワールドの圧倒的な没入感が初回で「このゲームは違う」印象を植える。無課金でも序盤が十分楽しい
// オンボーディング設計の失敗パターン
⚠️
チュートリアルが長すぎる
30分以上の強制チュートリアルは離脱の最大要因。特にモバイルでは最初の5分で継続か離脱かが決まる
⚠️
初回ガチャがしょぼい
最初の無料ガチャで「ハズレ」体験は致命的。初回はSSR確定か、それに近い体験を設計すべき
⚠️
メインループへの到達が遅い
ゲームの核心体験（育成・対人・探索）に辿り着くまでが長いと、本当の魅力が伝わらないまま離脱
⚠️
課金訴求のタイミングが早すぎる
愛着が生まれる前の課金誘導は反感を買う。最初の感情ピーク（キャラに愛着が湧いた後）まで待つべき

// フック設計の原則
01
複数フックの重層設計
日次フックだけでは飽きる。日・週・月のフックを重層的に組み合わせることで、プレイヤーが「今日やること」を常に持てる状態を維持する
02
フック密度の最適化
フックが多すぎると「やらなければならない」義務感が疲弊を生む。ライトユーザーには日次フックのみ、ヘビーユーザーには追加フックを用意する多層設計が理想
03
損失回避 vs 報酬獲得
「今日やらないと損」（損失回避）と「今日やると得」（報酬獲得）のバランスが重要。損失回避が強すぎるとストレス→離脱に繋がる
04
非リアルタイム設計の優位性
NIKKEの「まとめて殲滅」のような8時間分の放置報酬をまとめて回収できる設計は、多忙なプレイヤーのストレスを軽減しつつ継続率を高める

// Growth & Stagnation Design

📈 成長設計・停滞設計

プレイヤーが「強くなっている」と感じ続けるための設計と、意図的な壁（停滞）によって課金意欲を高める設計の両面を分析する。

// 成長曲線の4パターン
直線成長型
ウマ娘・学マス
1周ごとに確実に強くなる体験。ランダム要素があっても平均的な進歩が感じられる。周回モチベーションが高い
リスク: インフレが早く、古いコンテンツが陳腐化しやすい
段階解放型
FGO・ヘブバン・ファンパレ
ストーリー進行に連動してLv上限が解放される。「先が読みたい」動機が「強くなりたい」動機と連動
リスク: ストーリー消化が義務になり、コンテンツ追加コストが高い
対人相対型
プリコネ・ブルーロック・eFootball
自分の強さは他プレイヤーとの相対評価。環境メタが変わるたびに「また強化が必要」という設計
リスク: インフレが止まらず、追いつけない離脱者を量産する
放置積み上げ型
ラストウォー・キノコ伝説・ドット勇者
ログインのたびに数値が積み上がる安心感。「ちゃんと成長している」可視化が継続を支える
リスク: 課金者との差が開きすぎると無課金の離脱を招く
// 意図的な「壁」設計
💰 課金転換の壁
特定のコンテンツで無課金では突破が難しい難易度を設定。「ガチャで強いキャラを手に入れれば突破できる」という動機設計
例: プリコネのクランバトル上位、FGOの高難度クエスト
⏳ 時間の壁
「今は育成が足りないが、もう少し続ければ突破できる」という先を見せる設計。継続動機になる
例: スタミナ不足・素材集め・育成時間
🎲 運の壁
ガチャの確率的壁。「引けば突破できる」期待感が課金を誘発。天井設計はこの壁の安全網
例: 全ゲームのガチャシステム
🏆 競争の壁
他プレイヤーとの差が可視化されることで「もっと強くならないと」というプレッシャーが発生
例: ランキング表示、PvP格差
// 停滞設計の注意点
✅
良い停滞：越えられそうな壁。「あと少し」感があり、課金・継続で解決できると感じられる。フラストレーションが動機に転化される
❌
悪い停滞：越えられない壁。課金しても解決できないか、課金量が多すぎて諦める。フラストレーションが離脱に繋がる
⚠️
見えない壁：なぜ強くなれないのかわからない状態。情報の非対称性が不満を生む。攻略情報のエコシステムが整備されているゲームは壁が「見える化」される

  • 200〜300連で必ず最高レアを入手できる
  • 「最悪いくら使えば確保できるか」が明確で課金ハードルが下がる

  • 特定キャラへの需要に直接応える仕組み
  • 推し確保の計画性が生まれ長期課金習慣につながる
  • NIKKE・ウィッシュガチャ（学マス・ファンパレ等）で採用

  • 引いた回数分ポイントが貯まり素材や凸アイテムと交換

  • NIKKE（マイレージチケット）が典型
  • キャラ本体ではなく凸素材との交換に留まる場合が多い
📋 後払い・選択型
  • 結果を見てから支払いを確定できるスキーム
  • 「出なかったら払わない」選択がジレンマを生み逆に課金率が上がる効果
  • シートに記載の「後払いガチャ」がこの類型
  • ウィッシュガチャ（N連目で指定キャラ確定）も同様の安心感を提供

  • 1万円あたりの引ける回数の違いが体感的なお得感を決定する

9軸の骨格

9軸の連動 HOW THEY INTERLOCK

9軸は独立しているのではなく、次のように 連動 している：

  1. プレイヤーは、ある インセンティブ によって動き始める
  2. その行為は、一定の サイクル の中で反復される
  3. その過程で 揺らぎ が現れ、理解や操作の更新が求められる
  4. それに対処する中で チャンク が形成される
  5. 学習と有能感が積み重なり、インセンティブの 粘度 が上がる
  6. 行為が身体化され、作品世界と同期すると 横エネ が生まれる
  7. 進行や物語によって、同じ行為の 意味変容 が起こる
  8. それら全体が束になって、作品固有の 没入構造 を作る
  9. その結果として、作品特有の 感情の核 が立ち上がる
上層 — エネルギー体 UPPER LAYER · ENERGY BODY

作品全体を貫く 基調状態。設計者が無意識に縛っている 5つのパラメータで、9軸の運動の方向を形作る。物語・演出・操作・音・間 ── これらすべてを同じ向きに揃える「縛り」。

TEMPERATURE

RHYTHM

テンポの速い遅い、規則的／不規則。プレイヤーの身体が刻む基本拍。

BREATHING

エネルギー体は 9軸の運動の方向を縛り、その総和が 感情の核として立ち上がる。エネルギー体は設計者の選択、感情の核はプレイヤーの受容結果。

9軸の各論 EACH AXIS
01
サイクル
CYCLE — 何を、どの粒度で反復しているか

ゲーム体験は、単発の出来事ではなく 反復の積み重ね でできている。重要なのは「何が繰り返されているか」を 時間粒度ごとに 分けて見ること。

瞬間サイクル（数秒）

攻撃する／回避する／調べる／選択する。最小ループ。

プレイサイクル（数十秒〜数分）

1戦闘／1探索／1パズル／1区画の攻略。まとまった単位。

フェーズサイクル（章・エリア）

新ルール → 応用 → 裏切り → 乗り越え。中期的な進行。

全体進行サイクル

発見 → 理解 → 再解釈／戦闘 → 成長 → 難敵／探索 → 帰還 → 再出発。

→ その作品は何を体験の基礎運動としているか を見る。

02
インセンティブ
INCENTIVE — なぜその行為を続けるのか

報酬やご褒美ではなく、プレイヤーを前に進ませる理由全般を含む。たとえば──

・発見したい／上達したい／続きが知りたい
・世界を理解したい／このキャラを見届けたい
・この場所に居続けたい／自分のやり方を試したい

予測インセンティブ（チャンク化由来）

仮説を当てたい・伏線を回収したい。情報の断片から世界の理解を組み立てる過程そのものが動機になる。ミステリー型・考察型・伏線回収型ゲームのコア。
例：Outer Wilds（22分の世界に何が起きているのか）／ Obra Dinn（誰が誰を殺したか）／ ダクソ系のロア解読 ／ 13機兵防衛圏 など。

→ 何がプレイヤーの行為を駆動しているか の整理。

03
コスト
COST — 行為の対価。インセンティブと対をなす

プレイヤーが行為に対して 支払うもの。HP・MP・時間・スキル消費・操作労力など。インセンティブが「行為を駆動する力」なら、コストは「行為に伴う摩擦・対価」で、両者の対関係が ゲームループの動力源 になる。

マクロのコスト ⇄ ミクロのコスト

マクロ：作品全体で支払う対価（プレイ時間・学習負荷・感情負荷）
ミクロ（リスク）：個別行為の選択肢ごとの対価。これが豊かだと「リスクリターンの選択」が成立する。

コストが揺らぎを生む

コストが課されると、プレイヤーは「思った通りにいかない」状態に直面する＝ 難度の揺らぎ。コストが揺らぎ全体を作るわけではない（知覚・意味・ストーリーの揺らぎは別源）が、コストは 難度の揺らぎの主源。

コストを乗り越える反復が身体化を生む

選択肢の中からリスクリターンを判断し、行為を反復することで 操作チャンク が形成される。反復の蓄積が身体化＝身体横エネへと繋がる。

→ インセンティブとコストの対関係 がゲームループの基本動力。

04
インセンティブの粘度
VISCOSITY — 動機がどれくらい深く長く残るか

同じ「続けたくなる理由」でも、持続の仕方は違う。

低粘度

短期的・瞬間的な気持ちよさ（演出・手触り・一時的な驚き）。

中粘度

数十分〜数時間を引っ張る動機（次のエリアを見たい・新武器を試したい）。

高粘度

作品全体を支える深い動機（世界を理解したい・この旅を見届けたい）。

05
揺らぎ
FLUCTUATION — プレイヤーの認識・行為を更新させるズレ

体験はスムーズなだけでは深まらない。「思った通りにいかない」「見え方が変わる」「意味が揺れる」が必要。

難度の揺らぎ

単純に難しい／通じない／対応を変える必要がある。

知覚の揺らぎ

見え方が変わる／注意の向け方が変わる／以前の理解がずれる。

意味の揺らぎ

同じ行為が別の文脈を持ち始める（移動が帰還になる、戦闘が儀式になる）。

ストーリーの揺らぎ

世界の意味がまだ確定していない状態。環境ストーリーテリング／断片的な会話／用語だけ出てくる／世界の仕組みが説明されない／キャラの意味深な発言／なぜこうなったか分からない世界。
プレイヤー側の心理：情報不足 → 仮説を立てる → 少し理解する → 仮説を修正する → また情報が出る、というループ。これが ストーリーのチャンク化を生み、予測インセンティブと文脈横エネの両方の起点になる。

下層メカ：予測性とズレの運動

揺らぎは「半分わかる」状態を作ることで、予測の運動そのものを駆動する。

「半分わかる」が継続を生む

完全に予測できると驚きが消え、まったく予測できないと期待が立ち上がらない。癖は見える、何を期待すべきかは見える、しかし全体像と帰結までは見えない ── この状態でプレイヤーは仮説を持ち、感情と関心を乗せられる。

良いズレ vs 悪いズレ

悪いズレ：これまでの学習を無意味にする（突然展開が変わる、既存理解がただ否定される）
良いズレ：予測を壊さず 上位更新 する（途中までは正しかったが、もっと大きな文脈があった）。これは学習の否定ではなく拡張。

設計の運動式

予測できるから賭けられる。
ズレるから更新したくなる。
更新されるから続けたくなる。

06
チャンク
CHUNKS — プレイヤーの中で形成されるまとまり（認知的圧縮）

揺らぎに対処する中で、行為や認識を まとまりとして 覚えていく。これがチャンク。チャンクは 単なる学習ではなく認知的圧縮：複数のプロセスを「ワンセット」として扱える状態にすることで思考負荷を下げ、フローを生む。

操作チャンク

入力や手順のまとまり（回避→反撃／ジャンプ→着地→攻撃）。

判断チャンク

認識と選択のまとまり（この敵には距離を取る／この部屋は周辺確認）。

意味チャンク

行為の意味づけのまとまり（移動は探索／戦闘は生存ではなく証明）。

下層メカ：リスクリターン選択がチャンク多様性を生む

同じ揺らぎでも、複数の対処選択肢がある。プレイヤーが選び取ることで 個別化されたチャンク が形成される。

流れ：揺らぎ → リスクリターン選択 → チャンク化 → 身体化

例：ELDEN RING のボス戦で「ローリング回避（低リスク低リターン）」「パリィ→致命（高リスク高リターン）」「召喚霊で囮（中リスク中リターン）」のうちから選ぶ。選択がプレイヤーごとに異なるチャンクを作り、「自分のスタイル」 として身体化される。

リスクリターンはインセンティブ⇄コストのミクロ版

マクロのインセンティブ⇄コストが「作品全体の動機と対価」なら、リスクリターンは「個別行為の選択肢ごとの動機と対価」。スケール違いで同じ概念。

設計品質の判断

リスクリターンが 豊か／意味のある 作品 → プレイヤーが選び、個別化されたチャンクが形成 → 主体性ある身体化 → 横エネ深い
リスクリターンが 薄い／単調 な作品 → プレイヤーは選ばない → チャンクが画一化 → 身体化が浅い → 横エネが立ち上がらない

07
横エネ
YOKO-ENE — 行為が世界の意味に接続されたときに発生する

この理論で 最も独自 な概念。プレイヤーの行為が単なる操作や攻略としてではなく、その世界の 営みや振る舞い として感じられる状態。横エネには2つの成分があり、強い作品はその両方を備える。

成立条件①（プレイヤー側）

反復や学習を通じて身体リズムができていること。

成立条件②（作品側）

その行為が世界や文脈の中で意味を持っていること。

この2つが重なると、行為が「ゲームのための操作」ではなく、その世界の中で自然に行っていることのように感じられる。

2成分：身体横エネ × 文脈横エネ

身体横エネ（PHYSICAL）

操作・リズム・戦闘・回避・パリィ など、メカニクス由来の同期。プレイヤーの入力と結果が即時に一致し、身体的に世界に触れている感覚。
例：DMC のスタイリッシュ評価、Celeste の成功体験、ダクソのパリィ。

文脈横エネ（CONTEXTUAL）

世界・物語・意味・存在 など、ストーリー理解由来の同期。行為に「何のために？」が結びつき、世界の中で意味のある振る舞いをしている感覚。
例：Outer Wilds の探索、Death Stranding の運送、ICO の手を引く動き。

ゲーム身体文脈
モンハン強弱

ダクソ強強
Expedition 33強中〜強
Outer Wilds弱最強

行為＝物語のレベル（Lv1〜Lv4）

文脈横エネの強度は「行為がどれだけ物語に接続されているか」で測れる。4レベルあり、上に行くほど深い。

Lv1: 行為が物語進行のトリガー

戦闘 → 勝つ → ムービー → 物語 という連鎖。行為は物語を「進める」だけ。
例：FF7R

Lv2: 行為が物語の状況を表す

戦闘中の行為が「生きる／仲間を守る／遠征する」など、物語の現在の状況そのものを表現する。
例：Expedition 33

Lv3: 行為そのものが物語のテーマ

移動する／運ぶ／登る／倒す ─ それ自体が物語のテーマと等しい。
例：Death Stranding（運ぶ＝繋ぐ）／ Shadow of the Colossus（倒す＝喪失）

Lv4: 理解すること自体が物語

世界を理解する行為そのものが物語と等しい。プレイヤーの認識の更新が物語の進行と完全一致する。
例：Outer Wilds（世界を理解する＝物語）

// 横エネ強度の仮説式

横エネ強度 ≒ 世界理解量 × 行為が物語に与える意味量

世界理解量はストーリーの揺らぎとチャンク化が生み、行為の意味量は行為=物語のLv で測れる。両者の積で文脈横エネが決まる。

下層メカ：同期5種（精密化）

前述の「身体/文脈」は粗いラベル。実際にはプレイヤーが世界と同期する経路は 5種 あり、これらの組み合わせで横エネの質が決まる。行為=物語Lv（深さ）と同期5種（広さ）は直交する2次元軸。

① 身体同期

操作・リズム・戦闘・回避・パリィ。身体的な入力と結果の即時一致。例：DMC、Celeste。
前述の「身体横エネ」と等価。

② 環境同期

地形・気候・光・空気感など、世界の物理に呼応する。
例：BotWの登攀／天候、Death Stranding の地形。

③ 社会同期

NPC・他プレイヤー・コミュニティとの関係性に呼応する。
例：ELDEN RINGのメッセージ、Animal Crossingの島民、Death Stranding の非同期マルチ。

④ 意味同期

世界・物語・テーマ・象徴に呼応する。行為が世界の意味を持つ状態。
例：Outer Wildsの探索、Shadow of the Colossusの登攀。
前述の「文脈横エネ」のコア。

⑤ 時間同期

クール／日没／季節／ループなど、世界の時間構造に身体リズムが揃う。
例：Outer Wildsの22分ループ、どうぶつの森のリアルタイム連動。

→ プレイヤーの行為が世界と同期しているか を、5種類の経路 × Lv1〜4 の 2次元評価 で測る。

08
意味変容
TRANSFORMATION — 同じ行為が別の意味を帯びる

作品が深いと感じられるとき、多くの場合、同じ操作・同じ構造が後半で違う意味を持ち始める。

・最初はただの移動 → 後半では帰還／巡礼
・最初はただの戦闘 → 後半では執着／救済
・最初はただの収集 → 後半では記憶の回収

→ 「開始時／中盤／終盤／変化のきっかけ」を整理する。

下層メカ：未完チャンク5種（予測インセンティブの母体）

継続を生むのは、自分の中に 未処理のチャンクが残っていること。意味変容は、未完チャンクが時間をかけて回収・更新される過程として捉えられる。

① 因果未完

「あれは結局何だったのか」
過去の出来事や言及の因果関係が未確定で、後で回収されるという期待。

② 感情未完

「この関係はどう着地するのか」
キャラクター間の関係性や感情の決着がまだ見えていない。

③ 構造未完

「この世界の本当のルールは何か」
世界の仕組みや法則がまだ理解しきれていない。

④ 行為未完

「この操作や戦術にはまだ先があるのではないか」
身体化したスキル・チャンクの先に、まだ未到達の領域がある感覚。

⑤ 意味未完

「自分が今していることの意味は何か」
行為そのものの意味が未確定で、後で立ち上がってくる予感。

強い継続設計は、小さなチャンクは回収しつつ、中くらいのチャンクは少し広げ、大きなチャンクは未完のまま残す。プレイヤーは「ある程度の満足」を得つつ、「より具体化された大きな問い」を抱え続ける。これが継続の張力。

09
没入構造
IMMERSION — 何が束になって没入を作るか（プレイヤーの内面化）

没入はメカニクス単体では説明できない。複数の要素が重なって成立する。

・行為の気持ちよさ／世界の空気／物語の推進力
・音楽の支え方／演出の温度
・プレイヤーの存在の置かれ方

没入構造 ⇄ 横エネ：内向き / 外向きの対

没入構造（この軸） = プレイヤーの 内面化。行為・認知・感情がプレイヤーの中で同期した状態（内向き）。
横エネ（07） = プレイヤーの行為が 世界と同期 した状態（外向き）。
両者は同じ現象の 2つの側面：プレイヤー側を見るのが没入構造、プレイヤー × 世界の関係を見るのが横エネ。

内面化のプロセス

メカニクス上の コスト とキャラの ハードル が一致 → 操作とキャラ反応にズレがない → プレイヤーの意図とメカのインセンティブが一致 → 「思考-操作-状況」がチャンク化 → 世界との距離感が縮まる。
可視化：選択理由が「効率」から「キャラ／物語」に移行、同じ状況での意思決定が速くなり「キャラらしい選択」が安定する。

→ なぜその作品に深く浸れてしまうのか を、プレイヤー内向きで見る。

10
感情の核
EMOTIONAL CORE — 作品をその作品たらしめる感情の中心

作品にはそれぞれ、中心となる感情がある。単一である必要はないが、必ず何かしらの核がある。

・発見のよろこび／試行の楽しさ／緊張と解放
・喪失と回復／旅の軽やかさ／世界への畏れ／他者との共鳴

下層メカ：跳ね返り感情の蓄積から核が立ち上がる

感情の核は最初から存在するのではなく、ループの中で繰り返し生じる「跳ね返り感情」が蓄積 することで、作品全体を貫く中心感情として立ち上がる。

跳ね返り感情とは

ゲームループ（インセンティブ → アクション → コスト → フィードバック）を通じて生じる感情。
基本：嬉しさ、達成感
高次：後悔、誇り、安堵、罪悪感、執着、敬意、決意 など

2種類の感情源

① メカニクスコントロールから生じる感情：成功／失敗／克服の喜び
② キャラクターに紐づいて生じる感情：共感／後悔／決意
両者が 同期 すると、プレイ体験の意味づけが強まる。

同期させる設計レバー

・操作の重さや反動をアニメ・SE・UIで表現（コストを身体感覚に落とす）
・選択の結果が世界に残り続ける（NPC態度・街の変化・装備の見た目）
・内省のトリガー（夢・手紙・独白）を用意し、感情を言語化させる
・装備やパラ振りを「数値強化」ではなく「価値観の表明」と結びつける（防御強化＝守る意志、魔法強化＝探究心 など）

蓄積から核へ

メカ感情とキャラ感情が同期した状態で ループが回り続ける と、跳ね返り感情がプレイヤーの認知を更新し、新しい行動を促進する 高次の感情ループ が成立。これが作品固有の 感情の核 として残る。

→ 各作品ページの 入口 になる重要な観点。エネルギー体（基調）が形作り、9軸の運動が支え、ループの跳ね返りが蓄積して立ち上がる。

観察補助レンズ SUPPLEMENTAL LENSES

9軸では拾いきれない作品もある。以下の3つは 骨格に並列するサブ観察軸 として追加で持っておく。各作品ページの任意フィールドとして埋められるようになっている（埋まっている作品でのみ表示）。

＋A
時間構造
TIME STRUCTURE — 線形 / 分岐 / ループ / 非順序

物語と体験の進行が、時間軸上でどう走っているか。Outer Wilds の22分ループ、NieR Automata の周回ごとの視点切替、13機兵防衛圏 の非順序ノベル、など、時間構造そのものが意味を生む作品の観察用。

＋B
空間構造
SPATIAL STRUCTURE — 開放 / 閉鎖 / 迷宮 / 接続

世界が物理的にどう組まれているか。Hollow Knight の有機的に繋がる迷宮、Dark Souls の縦に重なる立体構造、Disco Elysium の都市の密度、BotW の開放と高低差、など、空間そのものが探索の感情を作る作品の観察用。

＋C
他者の在り方
PRESENCE OF OTHERS — NPC / マルチ / 非同期 / すれ違い

世界に他のキャラクター・他のプレイヤーがどう存在しているか。Death Stranding の非同期マルチによる「誰かがここを通った」感覚、ELDEN RING の召喚と侵入、動物の森 の島民の生活感、ICO の手を引く同伴、など、他者の存在そのものがコアになる作品の観察用。

＋D
移動メカニクス
MOVEMENT — 探索チャンク / 行為チャンク / 文脈性

移動はジャンルの骨格を決定する基本行動。どの種類のチャンクで深掘りするかで体験構造が変わる。

移動の5チャンク分類

行為チャンク：移動そのものの操作快楽（ARPG型）
成長チャンク：移動範囲の拡張＝キャラ成長（メトロイドヴァニア型）
環境チャンク：地形・気候との対話（BotW型）
構造チャンク：空間そのものの解読（Hollow Knight、Dark Souls型）
物語チャンク：移動が物語の出来事になる（Death Stranding、ICO型）

RPG vs ARPG の構造差

従来RPG：移動とバトルが 分離（ターン制の二層構造）。移動が探索チャンクの場、バトルが行為チャンクの場。
ARPG：移動とバトルが 連続包含。移動の中に行為チャンクが乗り、リズム同期が破綻なく流れる。

移動の3層

操作層：入力・反応の即時性
感情層：移動中に感じる気分・テンポ
意味層：移動が世界の中で何を意味するか（巡礼？逃走？運送？）
3層が同時に立ち上がる時、移動が 環境同期 + 意味同期 として横エネを生む。

→ 移動が深く設計されている作品は、それだけで没入の主戦場になる。

下層 — ゲームループ4要素 LOWER LAYER · GAME LOOP

9軸の運動を駆動する 動力源。インセンティブ → アクション → コスト → フィードバック の 1回転 がプレイヤーの体験のミクロな単位。これが反復することで、9軸のすべての運動（揺らぎ、チャンク化、身体化、横エネ、感情の核）が立ち上がる。

01 INCENTIVE
インセンティブ
行為を始める動機。「あの先に何かある」「もっと上手くなりたい」など。
02 ACTION
プレイヤーアクション
選び取った行為。リスクリターンの選択を含む。
03 COST
コスト
行為の対価。HP・MP・時間・操作労力。難度の揺らぎを生む主源。
04 FEEDBACK
フィードバック
結果の返り。跳ね返り感情（達成感／後悔／執着）を生む。

この4要素ループの 1回転がチャンク化の最小単位。何度も回ることで、操作チャンク → 判断チャンク → 意味チャンクへと身体化が深まり、跳ね返り感情が積み重なって 感情の核 が立ち上がる。

9軸との対応：インセンティブ=02軸、コスト=03軸、アクション=01サイクル、フィードバック=10感情の核 にそれぞれ繋がる。リスクリターン選択は、ループの「アクション」ステップ内に組み込まれる。

中心理論 — 揺らぎから横エネへ CORE THESIS

9軸を貫く 中心理論。揺らぎ・チャンク・意味変容・横エネ・感情の核 を一本の流れで結ぶ。

断片情報（揺らぎ） 　　↓ 仮説を立てる（予測インセンティブ） 　　↓ リスクリターンの選択肢が現れる 　　↓ プレイヤーが選び取る（主体性の発生） 　　↓ 選択がチャンク化される（認知的圧縮） 　　↓ 反復で身体化される 　　↓ 行為が世界文脈に接続（意味変容） 　　↓ 横エネ（行為が世界の意味と同期） 　　↓ 跳ね返り感情の蓄積 　　↓ 感情の核 が立ち上がる

つまり：揺らぎが 予測インセンティブ を生み（未完チャンクが母体）、リスクリターンの選択がチャンク化を 個別化 し、反復で 身体化 される。身体化された行為が世界の意味と接続して 横エネ を発生させ、ループを通じて 跳ね返り感情 が蓄積する。それが作品固有の 感情の核 を成立させる ── 9軸＋コスト軸はこの流れの各ステージを切り取った観察軸。

横エネとは、プレイヤーの行為が世界の意味に接続されたときに発生する。

ゲーム設計で考えるべき問い：
① プレイヤーは世界の何を理解していくのか？
② その理解によって行為の意味は変わるか？
③ 行為そのものに物語的意味はあるか？
④ 行為＝物語進行 で止まっていないか？（Lv1止まり）
⑤ 行為＝物語の出来事 になっているか？（Lv2以上）
⑥ 行為＝テーマ になっているか？（Lv3以上）

// IP × ソシャゲ EVALUATION

IP評価 / 横エネ自走

IPを「人気がある／売れている」結果論ではなく、長期的に感情が循環し続ける構造体として捉え直す。各タイトルのIPがどれだけ 横エネ自走可能 な構造を持つか、ソシャゲ化との噛み合いはどうかを評価する。

成功するIPとは、横エネルギーが自走可能な構造を持つIPである。

ここでいう「成功」とは、一時的なヒット・話題性・消費速度の速さではなく、ファンがその世界・キャラクターと 「居続けること」を選び続ける 状態。
そして ソシャゲの役割 は、IPに内在する横エネルギーを 反復・運用可能な形に変換するインフラ。新しい物語を語る装置でも、世界観の博物館でもない。

// 成功IPの3要件
①
世界が「振る舞い」で理解できる
「この世界なら、こう動くだろう」「このキャラなら、こうするはず」── 設定理解ではなく身体的な予測可能性。世界が 説明される対象 ではなく 呼吸を合わせられる環境 になっている。
②
キャラが「感情の置き場」
性格や設定の深さではなく、感情を預け続けても壊れない存在。安心して期待できる／待ち続けても裏切られない／裏切られても物語として受け止められる。
③
未完であることが前提
すべてが語られない／関係性が確定しきらない／終わりが最初から見えない。未完チャンクを抱えたまま滞在することが許される世界。この未完性が横エネを長期循環させる。
// 縦エネルギー vs 横エネルギー

成功IPは、縦エネルギーで惹きつけつつ、横エネルギーが自走する構造を持っている。

縦エネルギー

  • 物語を進める
  • 意味を回収する
  • 成長・達成・完結

→ 一読・一周で消費される性質

横エネルギー

  • 世界と同期する
  • 関係性に滞在する
  • 未完のまま居続ける

→ 反復・運用に堪える性質

// KPI観点 vs 感情反復観点 — ソシャゲがやるべき変換

実務上、ソシャゲは キャラ投入 / イベント投入 / 報酬投入 の3レバーで運用される。これらを KPI観点（結果の管理指標）として読むのと、感情反復観点（IPの横エネをどう運用可能化するか）として読むのは別物。

レバーKPI観点（結果管理）感情反復観点（設計の出発）
キャラ投入 新性能を売る 新しい感情の「居場所」を増やす
イベント投入 石を配る その感情に滞在する時間を設計する
報酬投入 行動を釣る 感情を裏切らないための補強
// 「借り物感」の正体

多くの日本IPは、物語完成度が高い／感情が原作内で回収される／横エネルギーが「消費型」で設計されている。
そのためソシャゲ化すると、縦エネの再消費・名シーン再演・設定の再提示 に終始しやすい。

IPが持つ横エネルギーの形式と、
ソシャゲが要求する横エネルギーの形式が噛み合っていない
── これが「借り物感」の正体。
// タイトル別IP評価

IP起点タイトル（GAMES.ip_type === 'IP起点'）について、横エネ自走可能性 / 借り物感 を評価。書き手が観察し直す前提で、現状はサンプル投入のみ。
